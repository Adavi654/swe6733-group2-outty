# Definition of Ready (DoR)

A Product Backlog Item is considered Ready when it contains:

## Required Elements

### 1. Title
A clear and concise description of the feature.

### 2. User Story
Must follow the format:

As a **user**,
I want **goal**,
So that **benefit**.

### 3. Additional Details
Acceptance criteria, technical notes, assumptions, or business rules.

### 4. Story Point Estimate
Estimated using Fibonacci sequence:

1, 2, 3, 5, 8, 13

### 5. Independent
The story can be completed within a sprint.

### 6. Understood
The development team understands the requirements.
